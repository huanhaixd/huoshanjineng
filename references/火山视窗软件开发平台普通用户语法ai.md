# 火山视窗软件开发平台普通用户语法

火山软件开发平台是一种**目的硬件设备无关**、**目的软件环境无关**、**易学易用实用**、**完全本地化**的软件快速开发平台。

火山平台采用独创的“**语言之上的语言**”（LOL）技术，成功屏蔽了火山程序对特定目标环境的依赖。用户只需要学习掌握火山程序语言，使用同一种语法就可以针对各种 不同目的环境进行软件开发。同时由于火山程序编译器所编译出来的是目标语言的源代码，因此能够最大限度地提升火山程序员与第三方的合作和资源共享效率， 以及所开发火山程序被社会的认可程度。

##### ● 关于火山安卓软件开发平台：

火山安卓软件开发平台由“**火山软件开发基础平台**”+“**支持安卓目标环境的火山类库**”+“**支持安卓目标环境的编译器/调试器插件及相关工具**”三部分组成，能够将用户开发的火山源程序编译为可以在安卓系统下运行的Java源程序。

 

##### ● 关于火山视窗软件开发平台：

火山视窗软件开发平台由**“火山软件开发基础平台”**+**“支持windows视窗系统目标环境的火山类库”**+**“支持windows视窗系统目标环境的编译器/调试器插件及相关工具”**三部分组成，能够将用户开发的火山源程序编译为可以在windows视窗系统下运行的C++源程序。 火山视窗支持编译**“32/64位”**的exe可执行文件和dll动态链接库，并全面支持Unicode(统一码/万国码）字符集。

 

##### ● 关于火山服务器软件开发平台：

火山服务器软件开发平台由“**火山软件开发基础平台**”+“**支持GO目标环境的火山类库**”+“**支持GO目标环境的编译器/调试器插件及相关工具**”三部分组成，能够将用户开发的火山源程序编译为对应的GO源程序 ，从而实现对跨平台服务器端软件的开发。

## 1.包名

```c
包名分类：
1.空包名
2.有名称的包名
```

### 空包名演示

```c
<火山程序 类型 = "通常" 版本 = 1 />

包 ""

```

### 有名称的包名

```c
<火山程序 类型 = "通常" 版本 = 1 />

包 火山.程序

```

```c
<火山程序 类型 = "通常" 版本 = 1 />

包 火山.大全.演示

```



### 共同特征

```c
1.空包名和有名称的包名都有固定格式
    格式如下：
 	<火山程序 类型 = "通常" 版本 = 1 />
2.有名称的包名可以自定义
    如下：
    包 火山.程序
    包 火山.大全.演示
3.不管是空包名还是有名称的包名，必须带有固定格式，否则在火山中是黏贴不了的
    固定格式如下：
    <火山程序 类型 = "通常" 版本 = 1 />
```

## 2.启动类

```c
启动类根据模板选择不同分为以下几种
1.空窗口程序
2.控制台程序
3.MFC窗口程序
4.WTL窗口程序
```

### 空窗口程序

格式如下：

```c
<火山程序 类型 = "通常" 版本 = 1 />

类 启动类 <公开 基础类 = 窗口程序类>
{
    方法 启动方法 <公开 类型 = 整数>
    {
        // 在此处填入自己的代码
        返回 (1)
    }

    #  
}

```

### 控制台程序

格式如下：

```
<火山程序 类型 = "通常" 版本 = 1 />

类 启动类 <公开 基础类 = 程序类>
{
    方法 启动方法 <公开 类型 = 整数>
    {
        // 在此处填入自己的代码
        返回 (1)
    }

    #
}

```

### MFC窗口程序

格式如下：

```c
<火山程序 类型 = "通常" 版本 = 1 />

类 启动类 <公开 基础类 = 窗口程序类>
{
    变量 主窗口对象 <类型 = 我的主窗口>

    方法 启动方法 <公开 类型 = 整数>
    {
        主窗口对象.创建主窗口 ()
        返回 (1)
    }
}

# ===

类 我的主窗口 <基础类 = 窗口 注释 = "样例主窗口" @视窗.布局 = "client_size = \"500, 300\"" 标题 = "我的主窗口">

```

### WTL窗口程序

### 他们的相同部分和不同部分

```c
相同部分
1.都有固定格式
    格式如下：
    <火山程序 类型 = "通常" 版本 = 1 />
2.都有一个相同关键字
    格式如下：
    类
    公开
    基础类
    方法
    类型
    返回
3.都有一个相同类名
    格式如下：
    启动类
4.都有相同的方法名
    格式如下：
    启动方法
5.都是相同的返回值
    格式如下：
    整数
6.都是返回1
    格式如下：
    (1)
7.都有花括号
    格式如下：
{
    
}
```

```c
不同部分
1.基础类的名称不一样
分别为：
空窗口程序的基础类名字为：窗口程序类
控制台程序的基础类名字为：程序类
MFC窗口程序的基础类名字为：窗口程序类
WTL窗口程序的基础类名字为：窗口程序类
2.MFC窗口程序和WTL窗口程序有其他两种程序没有分部分
格式为：
	主窗口对象.创建主窗口 ()
	类 我的主窗口 <基础类 = 窗口 注释 = "样例主窗口" @视窗.布局 = "client_size = \"500, 300\"" 标题 = "我的主窗口">
```

## 3.创建类

### 空类

```c
1.创建一个空类
格式如下：
<火山程序 类型 = "通常" 版本 = 1 />

类 ""
{

    #
}

```

```c
2.创建一个空的又是公开的类
    格式如下：
<火山程序 类型 = "通常" 版本 = 1 />

类 "" <公开>
{

    #
}

```

### 空类相同部分

```c
1.都是有固定属性标签，格式如下：
<火山程序 类型 = "通常" 版本 = 1 />
2.都有相同的花括号，格式如下：
{
    
}
3.在花括号中间都有一个固定符号，格式为
    #
4.都有固定关键字，格式为：
    类
5.都有空双引号，格式为：
    ""
```

### 有类名的类

```c
1.创建一个自定类名，自定义类名为：我是演示类
格式如下：
<火山程序 类型 = "通常" 版本 = 1 />

类 我是演示类
{

    #
}

```

```c
2.创建一个自定类名，但需要有公开关键字，自定义类名为：我是公开类
格式如下：
<火山程序 类型 = "通常" 版本 = 1 />

类 我是公开类 <公开>
{

    #
}

```

### 有类名相同部分

```c
1.都有固定标签，格式为：
<火山程序 类型 = "通常" 版本 = 1 />
2.都有固定关键字,格式如下：
    类
3.都有花括号，格式如下：
{
    
}
4.在花括号内部都有一个固定符号，格式如下：
    #
```

### 有类名不同部分

```c
1.有的类是公开有的是不是公开的，如有的类上面有公开两字就是代表类是公开的
2.自定义类名不同，如下：
    我是演示类
    我是公开类
```
